# Система складского учёта МТЗ АО «Фармцентр ВИЛАР»

Информационная система учёта складских операций с материально-техническими запасами и сырьём между отделами фармацевтического предприятия.

**Стек:** Python 3.11, FastAPI, PostgreSQL 15, SQLAlchemy, Alembic, React 18, Vite, Ant Design, Docker Compose, nginx, JWT, pytest, GitHub Actions

---

## Быстрый старт

Требуется **Docker**. Установка вручную не надобна.
Перед запуском сгенерируйте SECRET_KEY:

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

и установите пароль для базы данных в .env.

```bash
cp .env.example .env  # настройка переменных (пароль, SECRET_KEY)
make dev              # поднять проект
make seed             # создать admin-пользователя
make seed-demo        # заполнить БД демо-данными
```

Dev-билд поднимается на `http://localhost:3000`, Swagger на `http://localhost:8000/docs`.

**Admin:** логин `admin`, пароль задаётся при `make seed` (рекомендуется сменить).

### Демо-данные (`make seed-demo`)

Создаёт полный набор данных для демонстрации всех разделов системы:

- **8 товаров** с номенклатурными кодами, адресами хранения и остатками (один с низким остатком - для отчёта)
- **3 категории / 6 подгрупп** номенклатуры
- **3 поставщика**
- **3 подтверждённые поставки** + 1 черновик
- **5 расходных операций** (март–май 2025, типы: требование и отбор проб)
- **2 открытые ЛЗК** (Цех АФС и Цех ГЛС) с лимитами и историей операций
- **3 заявки** с разными статусами: ожидает / одобрена / отклонена
- **6 пользователей** по всем ролям системы

| Логин | Роль | Пароль |
|-------|------|--------|
| `omts_user` | ОМТС | `Demo1234!` |
| `afs_worker` | Цех АФС | `Demo1234!` |
| `gls_worker` | Цех ГЛС | `Demo1234!` |
| `okk_worker` | ОКК | `Demo1234!` |
| `ook_worker` | ООК | `Demo1234!` |
| `planner` | Плановый отдел | `Demo1234!` |

Скрипт идемпотентен - повторный запуск ничего не сломает.

## Переменные окружения

`.env.example` содержит все переменные с комментариями. Обязательно поменять:

- `POSTGRES_PASSWORD` - пароль БД
- `DATABASE_URL` - строка подключения (должна совпадать с паролем выше)
- `SECRET_KEY` - секрет для JWT (минимум 32 символа)

## Команды

```bash
make help # показать все команды
```

### Разработка

```bash
make dev # запуск проекта (hot reload, все порты открыты)
make down # остановка контейнеров
make logs # логи в реальном времени
make shell # bash внутри backend-контейнера
make migrate # применить миграции вручную (обычно не нужно — entrypoint делает это автоматически)
make seed      # создать admin-пользователя (первый запуск)
make seed-demo # заполнить БД демо-данными (товары, поставки, ЛЗК, заявки)
```

### Тесты

```bash
make test # полный прогон с coverage (изолированная БД, Docker-only)
make test-fast # быстрый прогон, стоп на первой ошибке
make lint # ruff lint
```

`make test` поднимает чистую тестовую БД в Docker, прогоняет тесты и сносит её.

### Прочее

```bash
make backup # резервная копия БД
make restore # восстановление из резервной копии
make install-hooks # установка pre-push хука (блокирует пуш при красных тестах)
make reset-password # сменить пароль пользователя
```

### Production

```bash
make prod # запустить в production-режиме (80 порт, no debug)
```

## Архитектура

```
[Браузер] -> :80 nginx -> /api/* -> backend:8000 (uvicorn)
                            └→ / -> статика React (dist/)
[БД] postgres:5432 - доступна только внутри docker-сети
```

В dev-режиме (`docker-compose.override.yml`) открыты все порты: 80, 8000, 3000, 5432.

## API

| Префикс | Описание |
|---------|----------|
| `/auth` | Авторизация, JWT, управление пользователями |
| `/references` | Справочники: категории -> подгруппы -> товары, поставщики |
| `/supplies` | Приход товара (приходные ордера) |
| `/expenses` | Расход товара (требования, отбор проб, списание) |
| `/requests` | Заявки от подразделений |
| `/limit-cards` | Лимитно-заборные карты (месячный цикл) |
| `/reports` | Карточка М-17, остатки, low-stock, дашборд |
| `/operations-log` | Журнал всех операций с фильтрацией и пагинацией |

Swagger UI (dev): `http://localhost:8000/docs`

## CI/CD

`.github/workflows/ci.yml` - 5 этапов на каждый пуш:

| # | Этап | Что делает |
|---|------|-----------|
| 1 | lint | ruff — статический анализ |
| 2 | security | pip-audit — сканирование CVE в зависимостях |
| 3 | test | `make test` — pytest с покрытием ≥ 85% |
| 4 | docker-build | сборка production-образов |
| 5 | deploy | SSH-деплой на `main`: backup -> сборка -> миграции -> перезапуск |
